## Project type

Website — marketing site for Acme Studio, an independent portrait photographer. Brochure + lead capture (contact form) + small editorial blog.

## Tech stack

- **Framework:** SvelteKit (Svelte 5)
- **Runtime:** Bun
- **CSS:** Token-driven CSS custom properties, cascade layers — see `docs/design-system/`
- **Forms:** Superforms + Valibot; contact form at `src/routes/contact/`
- **CMS:** Sveltia CMS (file-based content at `static/admin`), backend `acme-org/acme-studio`
- **Database:** Postgres + Drizzle — active. Runtime DB is `acme_studio` on the dev host; prod credentials live in `secrets.yaml`.
- **Deployment:** Podman + Caddy on `acme-prod-1` (Hetzner)

## Package manager — Bun only

This project uses **Bun**. Never substitute npm, npx, pnpm, or yarn.

```bash
bun install              # install dependencies (use --frozen-lockfile in CI)
bun add <pkg>            # add a package
bun run <script>         # run a script from package.json
```

**Never commit:** `node_modules/`, `.svelte-kit/`, `build/`, `dist/`, `.env`, `.env.*`, `bun.lockb`
**Always commit:** `bun.lock` (text lockfile)

- Do not add a second form without first checking the lead-routing workflow in n8n.

## Observability and error handling

Full docs: [docs/observability/README.md](docs/observability/README.md)

This project uses the template's **Tier 1** observability spine (small site, single-host deploy). No Sentry, no OpenTelemetry. The /healthz and /readyz probes are wired into the Caddy `health_uri` directive and into Healthchecks.io for daily liveness pings.

## Automation (n8n)

n8n runs on the same host as the site. Two workflows are active:

- **acme-contact-routing** — consumes `automation_events` of type `contact.submitted`, sends a Postmark email to `hello@acme-studio.dev`, opens a deal in the studio's CRM.
- **acme-content-publish** — opens a PR against `acme-org/acme-studio` when the studio publishes a new article from a third-party tool (Notion).

Webhook URL and secret live in `secrets.yaml` (`N8N_WEBHOOK_URL`, `N8N_WEBHOOK_SECRET`). Do not commit real values.

## Deployment

- Deployed to: `acme-prod-1` (Hetzner) via `git push main` → GitHub Actions → GHCR image → SSH `deploy.sh`
- Container image: `ghcr.io/acme-org/acme-studio:${SHA}`
- Caddy serves `acme-studio.dev` with TLS via `acme.zerossl.com`. The Caddyfile lives at `~/Caddyfile` on the prod host (templated from `deploy/Caddyfile.example`).
- Quadlet unit: `~/.config/containers/systemd/acme-studio.container` (templated from `deploy/quadlets/web.container`).
- Daily backup timer pushes `pg_dump` + `static/uploads/` tar to Cloudflare R2 (`r2:acme-studio-backups`).

## Project-specific rules

- The `/services` page is a static YAML collection, not a Postgres-backed listing. Do not introduce a `services` table.
- Articles must have a feature image; `og_image` falls through to it automatically (template default — do not override per-article unless the brand owner asks).
- The home hero image is the LCP element. Do not lazy-load it. Do not change `loading="eager" fetchpriority="high"` on `src/routes/+page.svelte`'s hero `<enhanced:img>`.
- `automation_events` of type `contact.submitted` must include `referrer` and `utm_*` query parameters when present. The lead-routing workflow needs them.

- `PUBLIC_GTM_ID=GTM-ACMEXYZ` (production GTM container)
- `PUBLIC_CLOUDFLARE_WEB_ANALYTICS_TOKEN` set in production for sanity layer

Tracked events: `contact_submitted`, `article_read`, `service_inquiry_started`. See `docs/analytics/event-taxonomy.md` for payload shape. Do not add new events without updating that file first.

## Privacy and retention

- `contact_submissions` retention: 18 months (studio retains lead history for booking follow-ups).
- `automation_events` retention: 90 days.
- `automation_dead_letters` retention: 30 days; alert on any row older than 7 days.

`bun run privacy:prune --apply` runs nightly via the systemd timer at `~/.config/systemd/user/acme-studio-prune.timer`.

## Backups

- `BACKUP_REMOTE=r2:acme-studio-backups` (Cloudflare R2)
- `BACKUP_HEALTHCHECK_URL` configured for nightly liveness pings
- Local backup retention: 7 days; remote retention: 90 days (lifecycle policy on the R2 bucket).

## What not to do

Standard template prohibitions apply. Project-specific prohibitions:

- Do not introduce a `services` database table — services are static content.
- Do not add a fourth font family.
- Do not lazy-load the home hero image.
- Do not change retention windows in `src/lib/server/privacy/retention.ts` without coordinating with the studio (regulatory implication for lead retention).
- Do not change Caddy's `health_uri` from `/healthz` — the prod uptime monitor depends on it.
